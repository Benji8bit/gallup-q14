# Руководство HR и администратора

## Доступ к pilot на interxion

| Параметр | Значение |
|----------|----------|
| URL опроса | https://<vps-host>:8443/survey |
| URL дашборда | https://<vps-host>:8443/admin |
| Пароль HR | `ADMIN_PASSWORD` на сервере (выдан при деплое) |

### Предупреждение браузера о сертификате

При **первом** открытии ссылки браузер может показать «Ваше подключение не защищено» — это нормально для pilot: используется временный self-signed сертификат, а не ошибка опроса.

**Что делать:**

1. Убедиться, что адрес — `https://<vps-host>:8443` (официальная ссылка от IT/HR).
2. Нажать **«Дополнительно»** → **«Перейти на сайт…»**.
3. После этого предупреждение обычно не повторяется в той же сессии браузера.

Для production планируется домен и сертификат Let's Encrypt (настраивает IT).

## Доступ к дашборду

1. Откройте `/admin` на хосте опроса
2. Введите пароль из переменной окружения `ADMIN_PASSWORD`
3. После входа откроется дашборд `/admin/dashboard`

> Пароль хранится в sessionStorage браузера как Bearer token. При выходе токен удаляется.

## Элементы дашборда

### KPI-карточки

| Карточка | Описание |
|----------|----------|
| **Индекс вовлечённости** | % favorable (4–5) по Q01–Q12. Дельта — изменение vs предыдущий квартал |
| **Участники** | Число уникальных отправок в текущем квартале |
| **Охват (штат компании)** | Доля участников от **актуального штата компании** на дату синхронизации Delivery (`v_employee` + `ods.employee`) |
| **Удовлетворённость Q00** | Средний балл 1–5 |
| **eNPS (E01)** | % промоутеров − % критиков; норма > +30 |

### Блок eNPS

Панель под графиками показывает три сегмента:

- **Промоутеры (9–10)** — «плюсы»
- **Нейтралы (7–8)** — не входят в формулу, но важны для анализа
- **Критики (0–6)** — «минусы»

### Графики

1. **Динамика по кварталам** — линейный тренд engagement %
2. **Radar по измерениям** — средний балл по 4 блокам ТЗ
3. **Bar chart по вопросам** — цвет по шкале Sapiens: синий ≥4, светло-синий ≥3, красный <3

### Рекомендации

Автоматически генерируются для 3 измерений с наименьшим % favorable. Используйте как отправную точку для квартального action plan, не как единственный источник решений.

## Синхронизация Delivery

**Dev:** кнопка **Delivery** пересобирает справочник из зеркала (`delivery_mirror.db`).

**VPS (pilot):** зеркала нет — кнопка **Delivery** переприменяет `delivery_reference_seed.sql` (агрегаты без PII). Свежие данные — monthly upload с машины в VPN.

Обновление зеркала из PostgreSQL — **раз в месяц**: `scripts/delivery-monthly-sync.ps1`.

### DBeaver (просмотр данных без VPN)

| Подключение | Файл |
|-------------|------|
| Delivery Mirror (Gallup Q14 local) | `backend/data/delivery_mirror.db` |
| Gallup Q14 app (local) | `backend/data/gallup-q14.db` |

Примеры SQL: `scripts/gallup_delivery_mirror_queries.sql`. Подробнее — [docs/02-setup.md — локальная копия Delivery](02-setup.md#локальная-копия-delivery-без-vpn).

Справочник **повторяет структуру Delivery** (`ods.employee` / `ods.v_employee`): все направления, должности и грейды как в источнике. Грейды не сворачиваются в Junior/Middle/Senior — в форме и срезах те же коды, что в Delivery (K1, DE3 и т.д.). Стаж — единственное производное поле (три корзины из `date_from`).

### Pilot (VPS)

На сервере: **`/opt/gallup-q14/scripts/delivery_reference_seed.sql`** — только агрегаты. Зеркало с email **не хранится**.

Обновление: `delivery-monthly-sync.ps1` на машине в VPN (pull зеркала → sync → export → `upload-reference-to-vps.ps1`).

## Экспорт CSV

Кнопка **CSV** на дашборде скачивает файл `gallup-q14-export.csv`.

Используйте для:
- Импорта в Excel / Power BI
- Архивирования результатов квартала
- Передачи в HRIS (при интеграции)

**Внимание:** CSV содержит `anonymous_token` — не публикуйте файл открыто.

## Квартальный процесс

### За 2 недели до конца квартала

- [ ] Согласовать даты кампании с руководством
- [ ] Разослать ссылку на опрос (корпоративный канал)
- [ ] Напомнить об анонимности и добровольности

### В период опроса (рекомендуется 2 недели)

- [ ] Мониторить число участников на дашборде
- [ ] Отправить напоминание при низком response rate

### После закрытия

- [ ] Экспортировать CSV
- [ ] Проанализировать рекомендации
- [ ] Подготовить презентацию для руководства
- [ ] Сформировать план мероприятий на следующий квартал

## Интерпретация показателей

| Engagement % | Интерпретация |
|--------------|---------------|
| ≥ 70% | Сильная вовлечённость |
| 50–69% | Зона внимания |
| < 50% | Критическая зона, нужен action plan |

Бенчмарки Gallup для крупных организаций — ориентир; для консалтинговой компании среднего размера фокус на **динамике** и **относительных различиях** между измерениями.

## Конфиденциальность

- Не пытайтесь деанонимизировать ответы по token или метаданным
- Не публикуйте разрезы по отделам при n < 5
- Храните CSV и доступ к дашборду только у уполномоченных HR

## Смена пароля администратора

1. Обновите `ADMIN_PASSWORD` в env
2. Перезапустите backend
3. Попросите всех админов войти заново
